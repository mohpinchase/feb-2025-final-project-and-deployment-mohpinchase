Essential Git Commands Every Developer Should Know

May 4, 2025 By Michael Chen 13 min read

Why Git Matters

Git is the most widely used version control system in software development. Understanding its core commands is essential for every developer. Let's explore the most important Git commands you'll use daily.

1. Repository Setup and Configuration

Initialize a Repository


# Create a new repository
git init

# Clone an existing repository
git clone https://github.com/username/repository.git

# Configure user information
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

2. Basic Commands

Track and Commit Changes


# Check status of working directory
git status

# Add files to staging area
git add filename.txt    # Add specific file
git add .              # Add all files

# Commit changes
git commit -m "Your commit message"

# Push changes to remote
git push origin main

3. Branching and Merging

Working with Branches


# Create and switch to new branch
git checkout -b feature-branch

# List all branches
git branch

# Switch branches
git checkout main

# Merge branch into current branch
git merge feature-branch

# Delete branch
git branch -d feature-branch

üí° Pro Tip

Always create a new branch for each feature or bug fix to keep your main branch clean and stable.

4. Inspecting Changes

View History and Changes


# View commit history
git log
git log --oneline    # Compact view

# View changes in working directory
git diff

# View changes in staged files
git diff --staged

# View changes in specific commit
git show commit-hash

5. Undoing Changes

Fix Mistakes


# Discard changes in working directory
git checkout -- filename.txt

# Unstage files
git reset HEAD filename.txt

# Modify last commit
git commit --amend

# Revert a commit
git revert commit-hash

# Reset to specific commit
git reset --hard commit-hash

‚ö†Ô∏è Warning

Be careful with reset --hard as it permanently discards changes. Make sure to commit or stash important changes first.

6. Remote Repository Operations

Working with Remotes


# Add remote repository
git remote add origin https://github.com/username/repo.git

# View remote repositories
git remote -v

# Fetch changes from remote
git fetch origin

# Pull changes from remote
git pull origin main

# Push changes to remote
git push -u origin feature-branch

7. Advanced Operations

Stashing and Rebasing


# Stash changes
git stash
git stash list
git stash pop

# Interactive rebase
git rebase -i HEAD~3

# Cherry-pick commits
git cherry-pick commit-hash

üí° Advanced Tip

Use interactive rebase to clean up your commit history before pushing to a shared repository.

Common Workflows

Feature Branch Workflow


# Start new feature
git checkout -b feature-x
# Make changes and commit
git add .
git commit -m "Add feature x"
# Push to remote
git push origin feature-x
# Create pull request
# After review and approval
git checkout main
git pull origin main
git merge feature-x
git push origin main

Best Practices

  • Write clear, descriptive commit messages
  • Commit early and often
  • Use branches for new features
  • Keep your main branch stable
  • Review changes before committing
  • Pull before pushing
